### 2.2 Рекомендательная система

**Текущее состояние:** ❌ НЕ РЕАЛИЗОВАНО

**Планируемая архитектура:**

```
┌─────────────────────────────────────────────────┐
│  🎯 ПЕРСОНАЛЬНЫЕ РЕКОМЕНДАЦИИ                   │
├─────────────────────────────────────────────────┤
│  Для тебя сегодня:                              │
│                                                  │
│  🎵 Концерт "Крым-Джаз"                         │
│     💡 Похоже на "Крым-Рок" из твоего избранного│
│     15 дек, 20:00 | ДК Севастополь              │
│                                                  │
│  🎭 Спектакль "Ревизор"                         │
│     💡 Ты часто смотришь театр                  │
│     18 дек, 19:00 | Театр Луначарского          │
│                                                  │
│  👶 Детский праздник                             │
│     💡 Популярно в Севастополе                  │
│     20 дек, 12:00 | ТЦ Муссон                   │
└─────────────────────────────────────────────────┘
```

**Алгоритмы рекомендаций:**

1. **Collaborative Filtering (похожие пользователи):**
   - Найти пользователей с похожими интересами
   - Рекомендовать события, которые они смотрели

2. **Content-Based (похожие события):**
   - Анализ избранного и истории пользователя
   - Рекомендации событий той же категории/места/артиста

3. **Популярность:**
   - Трендовые события в городе пользователя
   - Самые добавляемые в избранное

4. **Геолокация:**
   - События рядом с пользователем
   - Популярные места в его районе

**Что нужно реализовать:**

- [ ] **Сбор данных:**
  - Логирование действий пользователя (просмотры, избранное, покупки)
  - Таблица `user_interactions` (user_id, event_id, action_type, timestamp)

- [ ] **ML модель (простая версия):**
  - Расчёт similarity между событиями (TF-IDF по описаниям)
  - Кластеризация пользователей по интересам
  - Ранжирование событий для каждого пользователя

- [ ] **API эндпоинт:**
  - `GET /api/recommendations/{user_id}` - топ-10 рекомендаций
  - Параметры: category, city, limit

- [ ] **Интеграция в бот:**
  - Кнопка "🎯 Рекомендации для меня"
  - Ежедневная рассылка топ-3 рекомендаций

## 6. МОНЕТИЗАЦИЯ

### 6.1 Динамическая модель ценообразования

**Концепция:**
Цена размещения рассчитывается автоматически по формуле на основе:
- Охвата аудитории (количество подписчиков)
- Геолокации (вся база, город, район)
- Времени до начала события (коэффициент срочности)
- Конверсии (настраивается в админке)

**Текущее состояние:**

```
┌─────────────────────────────────────────────────┐
│  ТЕКУЩИЙ ФУНКЦИОНАЛ                             │
├─────────────────────────────────────────────────┤
│  ✅ Таблица placements в БД                     │
│  ✅ Админ-панель для управления размещениями    │
│  ✅ Статусы: pending→approved→paid→active       │
│  ❌ Динамический расчёт цены                    │
│  ❌ Геолокация пользователей                    │
│  ❌ Настройки в админке (x, y, Q)               │
│  ❌ User Flow (выбор тарифа)                    │
└─────────────────────────────────────────────────┘
```

---

#### 🎯 Способы продвижения

**1. Рассылка всем пользователям**
- **Описание:** Уведомление отправляется всем пользователям бота
- **Аудитория:** `r` (все подписчики)
- **Формула:** `((x * r) * y) * Q`
- **Пример:** 1₽ × 10,000 × 5% × 2 = **1,000₽**

**2. Рассылка по городам**
- **Описание:** Уведомление пользователям из определённого города
- **Аудитория:** `r_sit` (подписчики, чей город известен)
- **Формула:** `((x * r_sit) * y_sit) * Q`
- **Пример:** 1₽ × 3,000 × 8% × 1.5 = **360₽**

**3. Рассылка по районам города**
- **Описание:** Уведомление пользователям из определённого района
- **Аудитория:** `r_sit_zone` (подписчики, чей город И район известен)
- **Формула:** `((x * r_sit_zone) * y_sit_zone) * Q`
- **Пример:** 1₽ × 800 × 12% × 2.7 = **259₽**

**4. Горячие события (🔥)**
- **Описание:** Размещение в разделе "Горячее" (события в течение 24 часов)
- **Особенности:**
  - Показываются только активные события с началом в течение 24ч
  - Сортировка по времени начала (ближайшие первыми)
  - События без времени → в конце с пометкой "В течение всего дня"
  - **Коэффициент Q = 1** (фиксированный)
- **Формула:** `((x * r) * y) * 1`
- **Пример:** 1₽ × 10,000 × 5% × 1 = **500₽**

---

#### 📐 Переменные и коэффициенты

**Базовые переменные:**

| Переменная | Описание | Пример | Где меняется |
|------------|----------|--------|--------------|
| `x` | Стоимость лида (₽) | 1₽ | Админ-панель |
| `r` | Всего подписчиков | 10,000 | Автоматически из БД |
| `r_sit` | Подписчики в городе | 3,000 | Автоматически из БД |
| `r_sit_zone` | Подписчики в районе | 800 | Автоматически из БД |

**Конверсия:**

| Переменная | Описание | Пример | Где меняется |
|------------|----------|--------|--------------|
| `y` | Общая конверсия (%) | 5% | Админ-панель |
| `y_sit` | Конверсия в городе (%) | 8% | Админ-панель |
| `y_sit_zone` | Конверсия в районе (%) | 12% | Админ-панель |

**Временной коэффициент Q:**

| Время до события (a) | Коэффициент Q | Логика |
|---------------------|---------------|--------|
| a < 2 часа | **3.0** | Максимальная срочность |
| 2ч < a < 6ч | **2.7** | Высокая срочность |
| 6ч < a < 12ч | **2.2** | Средняя срочность |
| 12ч < a < 24ч | **2.0** | Завтра |
| 24ч < a < 30ч | **1.8** | Послезавтра |
| 30ч < a < 36ч | **1.5** | Через 1-1.5 дня |
| 36ч < a < 48ч | **1.3** | Через 2 дня |
| a > 48ч | **1.0** | Более 2 дней |
| "Горячие" | **1.0** | Всегда фиксированный |

**Логика:** Чем ближе событие, тем выше коэффициент (срочность = премиум цена)

```python
    """
    Рассчитывает цену размещения по формуле:
    Цена = ((x * r) * y) * Q
    """
    # Получить настройки из БД
    settings = get_monetization_settings()
    x = settings['cost_per_lead']

    # Получить размер аудитории
    if placement_type == 'broadcast_all':
        r = get_total_users()
        y = settings['conversion_general']
    elif placement_type == 'broadcast_city':
        r = get_users_by_city(city)
        y = settings['conversion_city']
    elif placement_type == 'broadcast_zone':
        r = get_users_by_zone(city, zone)
        y = settings['conversion_zone']
    elif placement_type == 'hot':
        r = get_total_users()
        y = settings['conversion_general']

    # Вычислить коэффициент Q
    if placement_type == 'hot':
        Q = Decimal('1.0')
    else:
        hours_until_event = (event_start_time - datetime.now()).total_seconds() / 3600
        Q = get_time_coefficient(hours_until_event)

    # Формула
    y_decimal = y / 100
    price = ((x * r) * y_decimal) * Q

    return {
        'price': round(price, 2),
        'audience_size': r,
        'conversion_rate': y,
        'time_coefficient': Q,
    }

def get_time_coefficient(hours: float) -> Decimal:
    """Определяет коэффициент Q по времени до события."""
    if hours < 2:
        return Decimal('3.0')
    elif hours < 6:
        return Decimal('2.7')
    elif hours < 12:
        return Decimal('2.2')
    elif hours < 24:
        return Decimal('2.0')
    elif hours < 30:
        return Decimal('1.8')
    elif hours < 36:
        return Decimal('1.5')
    elif hours < 48:
        return Decimal('1.3')
    else:
        return Decimal('1.0')
```
