## 1. ОПИСАНИЕ ПРОЕКТА

### 1.1. Общая концепция

**"Куда пойти"** — это готовое white-label решение (Telegram-бот + админ-панель), которое продается владельцам региональных Telegram-каналов (50k-1M подписчиков) **единоразово**.

После покупки владелец канала получает:
- ✅ Полностью настроенный бот под его брендом
- ✅ Админ-панель для управления контентом и рекламой
- ✅ Автоматический парсинг бесплатных мероприятий
- ✅ Инструменты монетизации (платные размещения)
- ✅ Полную независимость (никаких комиссий вам)

### 1.2. Уникальное торговое предложение (УТП)

#### Для ВАС (разработчик):
- **Быстрая окупаемость**: 1 продажа = 150-300k₽
- **Масштабируемость**: каждый регион = новый клиент
- **Пассивный доход**: опционально продавать техподдержку (10k/мес)

#### Для ВЛАДЕЛЬЦА КАНАЛА (клиент):
- **Монетизация канала**: зарабатывай на платных размещениях
- **Автоматизация**: парсинг мероприятий без ручного труда
- **Ценность для аудитории**: удобный поиск событий в Telegram
- **Полный контроль**: сам устанавливаешь цены, без посредников

### 1.3. Отличие от конкурентов

| Критерий | KudaGo / Timepad | Ваше решение |
|----------|------------------|--------------|
| **Платформа** | Веб/приложение | Telegram (где уже есть аудитория) |
| **Монетизация** | Комиссия 15-25% с билетов | Владелец сам устанавливает цены |
| **Контент** | Только платные площадки | Авто-парсинг + платные размещения |
| **Контроль данных** | У платформы | У владельца канала |
| **Стоимость входа** | Бесплатно, но комиссии | Разовая покупка, 0% комиссий |

### 2.2. Монетизация владельца канала (клиент)

После покупки владелец канала зарабатывает на:

#### 1. Платные размещения мероприятий

**Модель ценообразования (примеры для владельца канала):**

**Вариант А: Фиксированная цена**
```
Организатор платит фикс-сумму за публикацию мероприятия

Примеры тарифов:
- Стандартное размещение: 2,000₽
- Топ-позиция (первые в поиске): 5,000₽
- Закрепление на 7 дней: 8,000₽
- Пакет "5 мероприятий": 15,000₽ (экономия 25%)
```

**Вариант Б: CPC (Cost Per Click)**
```
Организатор платит за каждый клик на кнопку "Билеты" / "Подробнее"

Примеры тарифов:
- CPC = 30₽ за клик
- Минимальный бюджет: 3,000₽
- Автоостановка при исчерпании бюджета
```

**Вариант В: CPM (Cost Per Mille)**
```
Организатор платит за 1000 показов мероприятия в результатах поиска

Примеры тарифов:
- CPM = 300₽ за 1000 показов
- Минимальный бюджет: 5,000₽
```

**Гибридная модель (рекомендуется):**
```
- Бесплатные мероприятия: автопарсинг, UGC (для наполнения базы)
- Платные мероприятия: организатор выбирает модель (фикс/CPC/CPM)
- Топ-размещения: фикс-доплата за приоритет
```

#### 2. Баннерная реклама

```
- Баннер в результатах поиска: 15,000₽/неделя
- Баннер в главном меню бота: 25,000₽/неделя
- Рекламный пост в канале через бота: 10,000₽
```

#### 3. Дополнительные услуги

```
- Рассылка по базе пользователей бота: от 20,000₽ (зависит от охвата)
- Аналитический отчет для рекламодателя: 5,000₽
- Эксклюзивная категория (только 1 организатор): от 50,000₽/мес
```

## 3. АНАЛИЗ КОНКУРЕНТОВ

### 3.1. Прямые конкуренты (веб-платформы)

| Платформа | Модель | Комиссия | Минусы для организатора | Ваше преимущество |
|-----------|--------|----------|-------------------------|-------------------|
| **KudaGo** | Партнерские ссылки на билеты | 5-10% с продажи | Сложная интеграция, нет прямого контакта с аудиторией | Простое размещение, прямой доступ к аудитории канала |
| **Timepad** | Продажа билетов | 3.9% + 49₽ за билет | Высокие комиссии при больших объемах | 0% комиссий, владелец канала устанавливает цены |
| **Afisha.ru** | Реклама + билеты | 10-20% | Дорого для малого бизнеса | Локальный охват, доступные цены |
| **Kassir.ru** | Билетный оператор | 15-25% | Очень высокие комиссии | Фикс-цена или CPC без % с продаж |

### 3.2. Непрямые конкуренты (Telegram-боты)

**Найденные решения:**
- RegToEvent, ScheduleBot — только для регистрации на события (не агрегаторы)
- Roofevents (Москва) — только собственные мероприятия

**Вывод:** Нет white-label ботов-агрегаторов для региональных каналов → голубой океан.

### 3.3. Почему владельцы каналов выберут вас?

1. **Низкий порог входа**: разовая покупка vs разработка с нуля (500k-1M₽)
2. **Быстрый запуск**: 1 день vs 3-6 месяцев разработки
3. **Проверенное решение**: готовые алгоритмы, парсеры, UI
4. **Монетизация "из коробки"**: не нужно думать как зарабатывать
5. **Независимость**: никаких комиссий сторонним платформам

## 4. ЦЕЛЕВАЯ АУДИТОРИЯ

### 4.1. Ваши клиенты (владельцы каналов)

**Профиль:**
- Владельцы региональных Telegram-каналов
- Аудитория: 50k - 1M подписчиков
- Города: Крым, Сочи, Краснодар, Казань, Екатеринбург, Новосибирск и др. (население 300k+)
- Тематика: городские новости, афиша, туризм, развлечения

**Боли:**
- Сложно монетизировать канал без потери аудитории (навязчивая реклама)
- Подписчики просят "куда сходить?" — нет удобного инструмента
- Конкуренты (другие каналы) начинают предлагать подобные сервисы
- Хочется автоматизировать публикации мероприятий

**Ценность вашего решения для них:**
- ✅ Новый источник дохода (платные размещения)
- ✅ Увеличение ценности канала для подписчиков (удержание аудитории)
- ✅ Автоматизация контента (парсинг мероприятий)
- ✅ Профессиональный инструмент без технических знаний
- ✅ Полный контроль над данными и ценообразованием

**Возражения и ответы:**

| Возражение | Ответ |
|------------|-------|
| "Дорого" | Окупается за 1-2 месяца при 15-20 платных размещениях |
| "Сложно настроить" | Мы настраиваем "под ключ" + обучаем за 2 часа |
| "А если не будет спроса на платные размещения?" | База наполняется автопарсингом → пользователи приходят → спрос появится. Первые клиенты могут быть со скидкой |
| "Можно ли попробовать бесплатно?" | Демо-версия на 7 дней или 50% предоплаты, остальное после запуска |

### 4.2. Клиенты владельца канала (рекламодатели)

**Профиль:**
- Организаторы мероприятий (концерты, спектакли, фестивали)
- Локальные бизнесы (кафе, рестораны, клубы, музеи)
- Туристические компании, экскурсоводы
- Event-агентства

**Боли:**
- Дорогая реклама на федеральных площадках (KudaGo, Timepad)
- Сложно привлечь локальную аудиторию
- Нет прямого контакта с потенциальными гостями
- Высокие комиссии билетных операторов

**Ценность для них:**
- ✅ Доступ к целевой локальной аудитории
- ✅ Гибкие тарифы (фикс/CPC/CPM на выбор)
- ✅ Прозрачная статистика (просмотры, клики)
- ✅ Низкие цены по сравнению с федеральными платформами

### 5.2. Основные модули

#### 5.2.1. Telegram Bot (для пользователей)

**Функции:**

1. **Поиск мероприятий**
   - Фильтры: дата (сегодня/завтра/выходные/дата), категория, бюджет
   - Геопоиск (если пользователь отправит геопозицию)
   - Сортировка: по дате, по популярности, по качеству

2. **Карточки мероприятий**
   - Изображение, название, дата, место, цена
   - Кнопки: "Билеты" (если платное), "На карте", "Поделиться"
   - Пометка "🔥 Рекламное" (если платное размещение)

3. **Поиск мест рядом**
   - Кафе, рестораны, бары (из парсинга 2GIS/Яндекс.Карты)

4. **UGC (пользователи предлагают мероприятия)**
   - Пользователь отправляет текст/фото/ссылку
   - LLM извлекает данные
   - Владелец модерирует через админку

5. **Избранное**
   - Пользователь может сохранить мероприятие
   - Уведомления за 1 день до события

**UI:**
- Inline кнопки (не более 6 на экран)
- Эмоджи для визуального выделения
- Краткие тексты (до 3 строк описания)

---

#### 5.2.2. Админ-панель (для владельца канала)

**Доступ:**
- URL: `https://yourdomain.com/admin`
- Логин/пароль (настраивается при установке)
- Роли: Admin (владелец), Moderator (помощник)

**Разделы:**

**1. Dashboard (главная)**
- Виджеты:
  - Активные пользователи (DAU/MAU)
  - Количество мероприятий (всего/активных/на модерации)
  - Доход (сумма платных размещений за месяц)
  - Топ-5 мероприятий (по просмотрам)
- Графики:
  - Динамика пользователей
  - Категории мероприятий (pie chart)
  - Статистика платных размещений

**2. Мероприятия**
- Таблица с фильтрами (дата, категория, статус, тип)
- Колонки:
  - Название, Дата, Категория, Источник
  - **Тип**: Бесплатное (парсинг/UGC) | Платное (от рекламодателя)
  - **Статус**: Активное | Прошедшее | Черновик
  - Просмотры / Клики
  - Действия (редактировать, удалить)

**CRUD:**
- Создать мероприятие вручную
  - Основные поля: название, дата, время, место, описание, изображение
  - **Тип**: Бесплатное | Платное
  - **Если платное**: выбрать рекламодателя, указать сумму оплаты, модель (фикс/CPC/CPM)
- Редактировать
- Удалить (с подтверждением)
- Bulk операции (массовое удаление, экспорт CSV)

**3. Платные размещения**

**Подраздел: Заявки на размещение**
- Таблица заявок от рекламодателей
- Колонки: Рекламодатель, Мероприятие, Тип (фикс/CPC/CPM), Бюджет, Статус
- Статусы:
  - **Pending** (на рассмотрении) → владелец одобряет/отклоняет
  - **Approved** (одобрено) → ждет оплаты
  - **Paid** (оплачено) → публикуется
  - **Active** (активно)
  - **Completed** (завершено)

**Действия:**
- Одобрить (переход в статус Approved)
- Отклонить (с причиной)
- Выставить счет (генерация PDF)
- Отметить как оплаченное (вручную или автоматически через webhook оплаты)

**Подраздел: Рекламодатели**
- CRUD для компаний-рекламодателей
- Поля: Название, Контакты, История размещений, Баланс (если предоплата)

**Подраздел: Прайс-лист**
- Настройка тарифов:
  - Фикс-цена: стандартное размещение (X₽), топ (Y₽), закрепление (Z₽)
  - CPC: цена за клик (X₽), минимальный бюджет (Y₽)
  - CPM: цена за 1000 показов (X₽), минимальный бюджет (Y₽)
- Публичная страница для рекламодателей (например, `yourdomain.com/advertise`)

**4. Модерация UGC**
- Очередь заявок от пользователей
- Для каждой:
  - Текст пользователя
  - Изображения (если есть)
  - AI-экстракция (что распознала система)
  - Действия: Одобрить | Одобрить с правками | Отклонить

**5. Аналитика**

**Пользователи бота:**
- Список (tg_id, username, город, дата регистрации, последняя активность)
- Статистика: новые за период, активные, churned

**Мероприятия:**
- Топ по просмотрам
- Топ по кликам
- Конверсия (просмотр → клик)

**Платные размещения:**
- Доход за период (день/неделя/месяц)
- Разбивка по типам (фикс/CPC/CPM)
- Средний чек

**6. Настройки**

**Общие:**
- Название бота, логотип, цвета
- Токен Telegram-бота
- Тексты (приветствие, помощь)

**Парсинг:**
- Включить/выключить источники (Afisha Goroda, Kassa24, Yandex Afisha)
- Расписание (частота парсинга)

#### 5.2.3. Модуль платных размещений (детально)

**Сценарий А: Фикс-цена**

```
1. Рекламодатель связывается с владельцем канала (Telegram, email, форма на сайте)
2. Владелец создает заявку в админке:
   - Выбирает/создает рекламодателя
   - Заполняет данные мероприятия (название, дата, место, описание, изображение)
   - Выбирает тип "Платное - Фикс"
   - Выбирает позицию: Стандарт (3000₽) | Топ (5000₽) | Закрепление (8000₽)
   - Указывает срок публикации (по умолчанию до даты мероприятия)
3. Система генерирует счет (PDF) с реквизитами владельца
4. Владелец отправляет счет рекламодателю
5. После оплаты владелец отмечает в админке "Оплачено"
6. Мероприятие публикуется в боте с пометкой "🔥"
7. Статистика доступна в разделе "Аналитика"
```

**Сценарий Б: CPC (Cost Per Click)**

```
1. Аналогично п.1-2, но выбирается "Платное - CPC"
2. Указывается:
   - Цена за клик (например, 30₽)
   - Бюджет (например, 5000₽ → макс. 166 кликов)
   - Ссылка для трекинга (куда ведет кнопка "Билеты")
3. После оплаты → публикация
4. При каждом клике:
   - Запись в БД (ad_clicks)
   - Уменьшение остатка бюджета (5000₽ - 30₽ = 4970₽)
   - Если бюджет исчерпан → автостоп
5. Dashboard рекламодателя (опционально): логин → статистика в реальном времени
```

**Сценарий В: CPM (Cost Per Mille)**

```
1. Аналогично, но "Платное - CPM"
2. Указывается:
   - Цена за 1000 показов (например, 300₽)
   - Бюджет (например, 6000₽ → 20,000 показов)
3. При показе в результатах поиска:
   - Запись в БД (ad_impressions)
   - Уменьшение остатка (на CPM/1000 за каждый показ)
4. Автостоп при исчерпании
```

**Важно:**
- Все платные мероприятия помечаются эмоджи "🔥 Рекламное" или "Реклама"
- Разделение в поиске: сначала топ платные → остальные по релевантности
- Опциональная модерация (владелец может запретить некачественный контент даже если оплачен)

   - "Вы публикуете мероприятия вручную → время"

   - Окупаемость за 1-2 месяца

   - Цена: 250k₽ "под ключ"
   - Включает: установку, обучение, 30 дней поддержки
   - Опционально: техподдержка 10k₽/мес

**Расходы:**
- Разработка (ваше время): условно 0₽ (bootstrapping)

- OpenAI API: 10k₽/год
- Маркетинг (реклама, контент): 50k₽
- **Итого расходы: ~120k₽**

**Чистая прибыль: ~3,500,000₽**

- 20 продаж × 300k₽ = 6,000,000₽
- Техподдержка: 15 клиентов × 10k₽ × 12 мес = 1,800,000₽
- **Итого: ~7,800,000₽**

## 13. КРИТЕРИИ ПРИЕМКИ MVP (для первой продажи)

### Функциональные

**Telegram Bot:**
- [ ] Пользователь может найти мероприятие (фильтры: дата, категория, бюджет)
- [ ] Карточки отображают всю информацию (фото, название, дата, место, цена)
- [ ] Платные мероприятия помечены "🔥 Рекламное"
- [ ] Кнопки работают (Билеты, На карте, Поделиться)
- [ ] UGC: пользователь может предложить мероприятие

**Админ-панель:**
- [ ] Владелец может войти (логин/пароль)
- [ ] Dashboard показывает метрики (DAU, события, доход)
- [ ] CRUD мероприятий работает (создать, редактировать, удалить)
- [ ] Платные размещения: одобрить заявку → выставить счет → отметить оплату
- [ ] Модерация UGC: очередь, одобрение/отклонение
- [ ] Аналитика: топ мероприятий, доход за период

**Автоматизация:**
- [ ] Парсеры собирают 100+ мероприятий/день
- [ ] Дедупликация работает (дубликаты не создаются)
- [ ] LLM-экстракция корректна (80%+ accuracy)

**Монетизация:**
- [ ] Фикс-цена: заявка → счет → оплата → публикация
- [ ] CPC: клик трекается, бюджет уменьшается, автостоп
- [ ] CPM: показы трекаются, бюджет уменьшается, автостоп

### Нефункциональные

**Производительность:**
- [ ] Бот отвечает < 2 секунд
- [ ] Поиск мероприятий < 500ms (без кэша)
- [ ] Админка грузится < 3 секунд

**Надежность:**
- [ ] Uptime > 99% (мониторинг через Sentry)
- [ ] Автоматический restart при падении контейнера

**Безопасность:**
- [ ] Пароли хешируются (bcrypt)
- [ ] HTTPS на всех endpoints
- [ ] Админка защищена JWT

**Юзабилити:**
- [ ] Админка работает на мобильных (responsive)
- [ ] Все формы имеют валидацию
- [ ] Кнопки имеют loading states

### Документация

- [ ] Руководство владельца (PDF, 20+ страниц)
- [ ] Видео-туториал (10-15 мин)
- [ ] Инструкция по установке
